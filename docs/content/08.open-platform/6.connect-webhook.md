# 开放平台 - 统计回调（Webhook）

::alert{type="info"}
本页面向生态合作方技术团队。
::

Webhook 用于回传配置行为结果，帮助你统计接入效果。

## 回调时机

常见状态：

- `success`：用户确认并配置成功
- `cancelled`：用户取消
- `error`：执行失败

## 请求示例

```http
POST https://your-domain.com/proxycast/callback
Content-Type: application/json

{
  "event": "connect",
  "status": "success",
  "relay_id": "your-relay",
  "ref": "campaign-2026",
  "timestamp": "2026-02-16T10:00:00Z"
}
```

## 字段建议

- `event`：事件类型
- `status`：状态值
- `relay_id`：服务方标识
- `ref`：渠道标记
- `timestamp`：事件时间

## 安全建议

1. 仅接受 HTTPS 回调
2. 校验来源与参数完整性
3. 对回调做幂等处理
4. 记录失败重试日志

## 监控建议

1. 统计成功率与取消率
2. 跟踪错误类型占比
3. 区分渠道来源效果
